# 证据分级

给每条进入排序的事实贴强度标签。来源一律来自本轮工具输出（web.search / web.search_news / data.get_fundamentals 等），不来自训练记忆。

## 三级 + 一态

- **强**：交易所 / 监管文件（财报、临时公告、问询函）、官方订单 / 中标 / 锁产能 / 预付款公告、业绩说明会记录、专利 / 标准 / 认证文件、项目审批记录
- **中**：可信财经媒体、行业刊物、行业协会数据、公司官网产品页、假设透明的研究分析、上下游公开披露交叉验证
- **弱**：KOL 帖子、社媒讨论、来源不明截图、无基本面佐证的价格异动——只能当线索，结论必须用更强来源确认
- **待验证**：重要但本轮工具没拿到 → 明说，并给确切核实路径（查哪个文件 / 哪个指标）

## 结论处理规则

- 确认事实与推断分开写；弱证据标明"线索"
- 不要把排序靠前的候选建立在纯弱证据上
- 每个最终候选至少：1 条强或中级来源支撑的卡位事实 + 1 条需求 / 产能 / 客户 / 财务质量事实 + 最缺的那块证明 + 最清晰的失效条件

## 降级红旗

出现以下情况下调置信度：
- 论点依赖单一未具名客户或一张截图
- 股价主要由社媒热度驱动
- 机会兑现成收入之前公司先要再融资
- 存货与应收增速持续快于收入
- 宣称稀缺但毛利率不见改善
- 管理层用主题词汇而分部数据没变化
