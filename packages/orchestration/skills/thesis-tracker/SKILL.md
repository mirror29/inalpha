# 论点跟踪（Thesis Tracker）

核心信念：**论点必须可证伪**——什么都推翻不了的"论点"只是信仰。证伪性证据要和支持性证据同样严格地记录。

## 数据纪律

- 用户带来的"新进展"先核实再入档：`web.search` / `web.search_news` 找来源，关键来源 `web.fetch` 读原文确认实际内容与日期
- 论点涉及的财务支柱（毛利扩张 / 现金流改善等）用 `data.get_fundamentals` 取当前读数，不凭记忆
- 股价 / 估值锚用 `data.get_ticker`、`data.get_bars`（默认 fresh）
- 公司名 → 代码用 `data.search_symbol`
- 催化剂日期只记"事件名 + 日期"，禁止把未发生事件展开成具体结论
- 每次更新标注数据截止时间；回复语言跟随用户

## 工作流

### 1. 建立论点（新标的）

向用户收齐（缺的引导补全，不要替用户编）：

- **标的**：名称 + 代码（`data.search_symbol` 核）
- **方向**：看多 / 看空（研究立场）
- **核心论点**：1-2 句话（例：「看多 X——产品结构转向软件带来毛利扩张 + 经营杠杆」）
- **支柱**：3-5 条支撑论据，每条尽量量化（"收入增速 >20%"而不是"增长不错"）
- **风险**：3-5 条**会推翻论点**的情形（不是泛泛的"市场风险"）
- **催化剂**：未来会验证 / 证伪论点的事件（财报、新品、监管决定）+ 预期窗口
- **估值参考**：论点兑现值多少（区间即可）；**退出条件**：什么情况认错离场

### 2. 记录新进展（更新时）

每个 data point 一行：日期 / 发生了什么 / 影响哪根支柱（强化 · 削弱 · 中性）/ 建议动作（不变 · 关注 · 复核论点）/ 更新后信心（高 · 中 · 低）。

财报类进展建议先走 earnings-analysis skill 做完整复盘，把结论作为 data point 回填。

### 3. 论点记分卡（每次输出都带）

| 支柱 | 原始预期 | 当前状态 | 趋势 |
|---|---|---|---|
| 收入增速 >20% | — | 最新季 22%（来源+日期） | 在轨 |
| 毛利扩张 | — | 同比持平 | ⚠ 落后 |

状态列的数字必须来自本轮工具输出并带来源。

### 4. 催化日历

| 日期/窗口 | 事件 | 预期影响 | 盯什么证据 |
|---|---|---|---|

### 5. 结论

当前信心等级 + 一句话理由 + 离"复核线/退出线"还有多远。**信心下调要和上调一样果断**——拖着不降级是最常见的论点腐烂方式。

## 档案与持久化

论点档案以结构化 markdown 块在对话中维护（开头标 `📋 Thesis: <标的> · vN · 更新于 <日期>`）；agent 记忆可跨轮引用，用户可随时要完整档案自行保存。用户管理多个标的时，主动提议做一轮全组合论点巡检。

## 边界

- 至少每季度复核一次，哪怕"什么都没发生"（什么都没发生本身可能就是负面证据——预期中的催化没来）
- 研究跟踪，不是交易指令：记分卡建议"复核论点"≠"卖出"；用户要据此调仓 → 走 `trade.create_plan` 审批链路
- 看空立场是合法的研究立场，但 Inalpha 模拟盘现货 long-only——看空论点的"动作"是回避 / 减仓 / 移出关注，不是开空单
