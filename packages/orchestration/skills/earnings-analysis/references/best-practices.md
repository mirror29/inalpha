# 财报复盘写法要诀

## 标题：必须带结论

好标题（事件驱动 + 有观点 + 量化方向）：
- 「X Q2：直营强劲抵消批发疲软——维持核心论点，上调全年收入预期」
- 「Y Q3：新品爬产超计划——上调明年出货预期」
- 「Z Q1：服务超预期、硬件不及——好坏参半，下调利润预期」

坏标题（无信息量，禁用）：
- 「X 公司季报更新」「公司发布财报」「Q3 业绩分析」

## 十条要诀

1. **结论先行**：beat 还是 miss？预期上调还是下调？第一句话给出
2. **量化一切**："表现强劲"没有信息量；"收入超共识 1.5 亿（+3%）"才有
3. **讲驱动不讲现象**：别只说"收入超预期"，说**为什么**超
4. **show the work**：旧预期 → 新预期，每条带原因
5. **听完业绩会再下结论**：管理层口径常比新闻稿多一层信息（transcript 用 web.fetch 读）
6. **同行对照**：同板块公司若已发布，给相对表现一句话
7. **克制篇幅**：这是季度更新不是首次覆盖，聚焦"新东西"
8. **指引变化必析**：给了指引就深挖，没给要明说
9. **时效即价值**：复盘基于"发布后最新可得"的材料；材料超 3 个月要警告用户
10. **连接论点**：结果必须落到"对投资逻辑意味着什么"，否则只是数字复述

## 常见错误

- 写成首次覆盖的长度（背景介绍超过两句话就是信号）
- 缺 beat/miss 定量（最常见也最致命）
- 不更新前瞻预期（只复述本季数字 = 没完成核心交付）
- 含糊语言（"较为强劲""有所改善"）
- 忽略指引、忽略股价反应
- 数字无来源、来源无日期、共识来源不明
- 把训练记忆里的旧季度当最新（开工四步没走）

## 交付前自检

- [ ] 标题带结论与方向
- [ ] 每个关键指标有 beat/miss 定量 + 原因
- [ ] 指引：有则新旧对比，无则明示
- [ ] 预期修正表：旧 / 新 / 变化 / 原因齐全（本年 + 明年）
- [ ] 论点影响明确（强化 / 削弱 / 中性，落到具体支柱）
- [ ] 股价反应已核（与分析矛盾时有解释）
- [ ] 每个数字可溯源：来源 + 日期 + 链接
- [ ] 结尾标注数据截止时间
