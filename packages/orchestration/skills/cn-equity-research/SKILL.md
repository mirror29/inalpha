# CN Equity Research · A股系统化调研方法论

核心纪律：**壁垒 → 增速 → 估值消化 → 资金 → 风险，从上到下逐层验证，任何一层
fail 就 stop，不跨层补救**。同一结论要多源交叉验证，不信单一指标。

## 数据纪律（最高优先级，覆盖本 skill 其余所有指引）

本 skill 只提供"怎么研究"的结构，**不含任何数据**。执行中所有"查"的动作必须落到工具：

| 方法论步骤 | 工具 |
|---|---|
| 公司名 → 可交易代码（一切个股研究的第一步） | `data.search_symbol`（A股返 sh./sz. 格式） |
| 实时价格 / 当日涨跌 | `data.get_ticker` |
| 历史 K 线（趋势 / 区间定位） | `data.get_bars`（保持默认 fresh） |
| PE_TTM / PB / ROE / 营收与利润增速 | `data.get_fundamentals`（venue=baostock） |
| 市场级财经快讯（盘面消息） | `data.get_market_news` |
| 行业板块涨跌幅榜（顺风 / 逆风） | `data.get_market_sectors` |
| 当日强势股 + 题材标签（主线确认） | `data.get_market_movers` |
| 沪深港通资金流向（市场级资金面） | `data.get_market_moneyflow` |
| 个股新闻 / 公告 / 研报线索 | `web.search_news`、`web.search` |
| **核一手来源原文**（公告 / 研报 / 新闻正文 + 日期） | `web.fetch` |
| 临时计算（PEG / 消化年限等公式） | `sandbox.run_code`（或心算后写明算式） |
| 深度多视角研究（决定要不要建仓级别） | `research.deep_dive`（本 skill 之后的下一步） |

**数据缺口的处理（硬规则）**：机构一致预期 EPS、研报评级、龙虎榜、限售解禁、
融资融券余额、股东户数**当前没有专用工具**——需要时用 `web.search` + `web.fetch`
找公开来源；找不到就在输出里**显式声明该维度数据缺失并跳过**。
**禁止用训练记忆代答任何具体数字 / 预测 / 评级**；所有引用数值标注数据时点（§3.1）。
回复语言随用户最近一条消息。

## 流程一 · 单票估值定位

1. `data.search_symbol` 解析代码 → `data.get_ticker` + `data.get_fundamentals` 拿
   现价 / 市值 / PE_TTM / PB / 增速
2. 找**前向估值**输入：机构一致预期 EPS（web 检索，找不到则只用 PE_TTM 并声明）
3. 计算三个核心指标（公式透明，算式写进输出）：
   - **PE_forward** = 现价 ÷ 明年一致预期 EPS（找得到预期时主用，比 PE_TTM 前瞻）
   - **PE 消化年限** = ln(PE_fwd ÷ 锚点) ÷ ln(1 + CAGR)——"以当前增速，多久消化到锚点估值"
   - **PEG** = PE_forward ÷ (CAGR × 100)
4. 判断标准（经验锚点，不是真理，输出时如实标注）：
   - 锚点 PE：全 A 经验值 ~30x；科创板属性可上调至 40-50x
   - PEG < 1.0 偏便宜 / 1.0~1.5 合理 / > 1.5 偏贵
   - 消化年限 1-2 年是理想窗口；> 3 年变量太多风险大

**"壁垒→增速→PE消化→PEG"决策链（fail 即 stop）**：
1. **壁垒**：技术壁垒（专利/算法）或产能壁垒？无壁垒 = 直接排除，再便宜也有淘汰风险
2. **增速**：CAGR < 20% 不该给高 PE；> 50% 才有资格承受 > 50x PE
3. **消化窗口**：见上
4. **PEG 终筛**：3、4 同时过关才进入候选

## 流程二 · 批量估值对比

同一赛道多只标的各跑一遍流程一，输出对比表（PE_fwd / PEG / 消化年限 / 数据完整度），
按 PEG 升序找"增速配得上估值"的标的。纪律：
- 优先有机构覆盖（找得到一致预期）的标的；找不到预期的小盘股标注"仅 PE_TTM，前瞻性弱"
- 一致预期分歧大（不同来源差 > 20%）→ 标注"机构分化，可能有未消化事件"，不要取均值装精确

## 流程三 · 新标的快速调研清单

按序执行，每步结论落到工具数据：

1. **基本面筛查**：`data.get_fundamentals`——市值是否过流动性门槛（经验值 ≥ 20 亿）、
   PE/ROE/增速基础画像
2. **估值定位**：流程一的三指标
3. **题材确认**：`data.get_market_movers` 的 tags 是否涉及该股 / 其所属概念；
   `data.get_market_sectors` 看所属行业当日排位（顺风 / 逆风）
4. **资金面**：`data.get_market_moneyflow`（市场级方向）；个股级资金流无工具——声明缺口
5. **风险预警**（web 检索，逐项找不到就声明）：限售解禁倒计时 / 融资余额趋势 /
   股东户数变化 / 近期龙虎榜
6. **信息流**：`web.search_news` 个股新闻频率与方向；关键公告用 `web.fetch` 读原文
7. **产业链定位**：上下游在板块榜的位置；要继续深挖产业链瓶颈 → 切 serenity skill

## 流程四 · 盘面/题材归因互证

orchestrator 已有六维归因框架（消息面/板块/题材/资金/宏观/技术面）；本节只补**互证规则**：

- **题材共识强**需三条以上同向：强势股 tags 聚类指向该题材 + 相关板块涨幅居前 +
  快讯里有对应消息 +（可得时）资金面同向
- 只有一条来源支持的"主线"按"单一来源，待证实"措辞
- 北向数据是估算口径（工具 note 已标注），只用于方向判断，禁止引用为精确净买入额

## 常见坑

- **数据滞后**：财务快照季报后才更新；股东户数滞后 3-6 个月；宁可用"最新的粗数据"
  也别把"准确的过时数据"当现状，重大结论前核对数据发布日期
- **口径差异**：不同源的市值 / 换手率定义不同（成交量÷流通股本 vs 成交额÷市值），
  对比时统一口径，不混用
- **ST 股**：常年亏损 PE 失效、PEG 无意义；解禁与退市风险突出；两融多被禁；
  筛选时默认排除，用户点名问才分析并强调风险
- **科创板**：涨跌停 20%、估值中枢高于主板、小标的常无机构覆盖——锚点和龙虎榜
  信号强度都要相应调整
- **北向资金**：交易所 2024-08 起停止盘中披露官方数据，任何"北向实时净买入"都是
  第三方估算——引用必须带口径声明

## 输出格式建议

单票评估用紧凑结构（占位符仅示意格式，不是预设标的）：

```
【<代码> <名称>】数据截至 <时点>
估值   PE_TTM <x> | PE_fwd <x 或"无一致预期">| PEG <x> | 消化年限 <n>年 → <便宜/合理/贵>
题材   所属板块当日 <+x%>（榜单第 <n>）| 强势股标签命中：<tags 或 无>
资金   市场级：北向 <方向>（估算口径）| 个股级：数据缺口
风险   解禁 <情况或"未查到"> | 两融 <情况或"未查到">
结论   <一句话研究优先级 + 证据链指向哪几个工具结果 + 缺了哪些数据>
```

批量对比用表格（PE_fwd / PEG / 消化年限 / 数据完整度 / 一句话评级）。
所有结论是**研究优先级**，不是买卖指令；要落到交易决策走 research.deep_dive + 回测链路。
